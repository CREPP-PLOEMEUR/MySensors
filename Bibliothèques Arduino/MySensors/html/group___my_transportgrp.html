Detailed Description

API declaration for MyTransport.

Transport-related log messages, format: [!]SYSTEM:[SUB SYSTEM:]MESSAGE

  • [!] Exclamation mark is prepended in case of error
  • SYSTEM:
    • TSM: messages emitted by the transport state machine
    • TSF: messages emitted by transport support functions
  • SUB SYSTEMS:
    • Transport state machine (TSM)
      • TSM:INIT from stInit Initialize transport and radio
      • TSM:FPAR from stParent Find parent
      • TSM:ID from stID Check/request node ID, if dynamic node ID set
      • TSM:UPL from stUplink Verify uplink connection by pinging GW
      • TSM:READY from stReady Transport is ready and fully operational
      • TSM:FAIL from stFailure Failure in transport link or transport HW
    • Transport support function (TSF)
      • TSF:CKU from transportCheckUplink(), checks connection to GW
      • TSF:SID from transportAssignNodeID(), assigns node ID
      • TSF:PNG from transportPingNode(), pings a node
      • TSF:WUR from transportWaitUntilReady(), waits until transport is ready
      • TSF:CRT from transportClearRoutingTable(), clears routing table stored in EEPROM
      • TSF:LRT from transportLoadRoutingTable(), loads RAM routing table from EEPROM (only GW/repeaters)
      • TSF:SRT from transportSaveRoutingTable(), saves RAM routing table to EEPROM (only GW/repeaters)
      • TSF:MSG from transportProcessMessage(), processes incoming message
      • TSF:SAN from transportInvokeSanityCheck(), calls transport-specific sanity check
      • TSF:RTE from transportRouteMessage(), sends message
      • TSF:SND from transportSendRoute(), sends message if transport is ready (exposed)
      • TSF:TDI from transportDisable()
      • TSF:TRI from transportReInitialise()
      • TSF:SIR from transportSignalReport()

Incoming / outgoing messages:

See here for more detail on the format and definitions.

Receiving a message

  • TSF:MSG:READ,sender-last-destination,s=%d,c=%d,t=%d,pt=%d,l=%d,sg=%d:%s

Sending a message

  • [!/?]TSF:MSG:SEND,sender-last-next-destination,s=%d,c=%d,t=%d,pt=%d,l=%d,sg=%d,ft=%d,st=%s:%s

Prepended char:

  • none=sending OK
  • !=error sending
  • ?=sending status unknown Message fields:
  • s=sensor ID
  • c=command
  • t=msg type
  • pt=payload type
  • l=length
  • sg=signing flag
  • ft=failed uplink transmission counter
  • st=send status, OK=success, NACK=no radio ACK received
